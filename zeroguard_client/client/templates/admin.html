OAuth 2.0 Flow This App Uses
1. Login → Client generates PKCE code_verifier + code_challenge
2. Redirect → Browser sent to IDP /oauth/authorize
3. Auth → User enters credentials + TOTP at IDP
4. Code → IDP redirects back with auth code
5. Exchange → Client POSTs code + code_verifier to /oauth/token
6. PKCE verify → IDP verifies SHA256(verifier) == challenge
7. Tokens → IDP returns access_token + id_token + refresh_token
8. Validate → Client verifies id_token using IDP's JWKS public key
9. Session → User is logged in, all API calls use access_token
